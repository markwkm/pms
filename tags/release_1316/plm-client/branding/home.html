Welcome to the Patch Lifecycle Manager

The Patch Lifecycle Manager (PLM) provides a central point for tracking the lifecycle of patches against a source tree.

Regardless of the RCCS system used by the developer to track development of the patch (CVS, Subversion, BitKeeper, or by hand...) The PLM can be used as a central publication point. This provides a community location for peer review and testing.

The required format for a patch is any one supported by the standard 'diff' and 'patch' utilities.

Metadata about the patch is stored and can be used to save information such as what regression tests the patch has passed or failed.

For a quick introduction to the PLM, please see the MINI-HOWTO.

This installation of the PLM is targeted at the Linux Kernel.

Once a patch is in this system, you can test it at the Scalable Test Platform.

Please send comments to the developers.
