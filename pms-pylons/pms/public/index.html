Welcome to the Patch Management System

The Patch Management System (PMS) provides a central point for tracking the lifecycle of patches against a source tree.

Regardless of the RCCS system used by the developer to track development of the patch (CVS, Subversion, or by hand...) The PMS can be used as a central publication point. This provides a community location for peer review and testing.

The required format for a patch is any one supported by the standard 'diff' and 'patch' utilities.

Metadata about the patch is stored and can be used to save information such as what regression tests the patch has passed or failed.

For a quick introduction to the PMS, please see the MINI-HOWTO.
